第一章 Java 入门

Java 跨平台的原理

Java 语言编写的源代码，是.java 文件，经过编译后，能够运行的是.class 文件。这些.class 文件不能直接在操作系统中运行，我们需要在不同的操作系统中安装相应的 Java 运行环境(JRE，Java Runtime Environment)，这个 Java 运行环境中包含了 Java 虚拟机(JVM，Java Virtual achine)，.class 文件在不同的 Java 虚拟机上运行即可。

类型转换

  1. 低精度的值可以直接赋值给高精度的变量，直接变成了高精度。

精度：byte

  1. 整型默认为int 即：“byte num = 10;”中10是int，系统隐含做了一个从 int 转向 byte，并不丢失精度的操作，如果不丢失精度，则不报错。上述说明仅仅局限于将常量赋值给变量的情况。当“byte num2 = num + 1;”时就会报错，在变量求值时，byte、short、char 类型的变量值被自动提升为 int 型，最后结果是int，需要强制转换，“byte num2 = (byte) (num + 1);”。

  2. 不同类型变量混合运算之后，得到的结果是精度最高的类型。

第五章 面向对象编程(一)

实例化对象

类名 对象名 = new 类名();

比如：

java

成员函数

Java 中的函数，编写在类中，一般称为成员函数。

如果函数内部的变量和类中的成员变量重名，成员变量被屏蔽。得不到正常效果。此时，可以用“this.”来标志该变量属于类中的成员，而不是局部变量。

函数重载

函数重载，也叫做静态多态。

多态(Polymorphism)，是面向对象编程的特征之一。多态，通俗讲就是一个东西， 在不同情况下呈现不同形态。比如，一个函数名 abs，在不同参数的情况下可以执行不同的代码，而调用者只需要记住一个函数名称。

为什么是静态的呢？那是因为虽然函数名只有一个，但是源代码中，还得根据不同参数编写多个函数。

函数重载必须满足以下条件之一：

  1. 函数参数个数不同。

  2. 函数参数个数相同，类型不同。

  3. 函数参数个数相同，类型相同，但是在参数列表中的出现顺序不同。

构造函数

构造函数也是一种函数，但是定义时必须遵循以下准则：

  1. 函数名称与类相同的名称。

  2. 不含返回类型。

  • 构造函数可以被重载，也就是说，一个类中可以定义多个构造函数。实例化对象时，系统根据参数的不同，调用不同的构造函数。
  • 当一个类的对象创建时，构造函数就会被自动调用，可以在这个函数中加入初始化工作的代码。在对象的生命周期中，构造函数只会调用一次。
  • 在一个类中，如果没有定义构造函数，系统会自动为这个类产生一个默认的构造函数，该函数没有参数，也不做任何事情。因此，我们在没有定义构造函数时，才可以通过“类名 对象名 = new 类名();”来实例化对象。

如果我们自己定义了含有参数的构造函数，系统将不提供默认的构造函数。

第六章 面向对象编程(二)

静态变量

定义成静态变量，方法是在其定义前加上“static”关键字即可。例如：

java

  • 静态变量可以通过“对象名.变量名”来访问，如：“zhangsan.bankName”，也可以通过“类名.变量名”来访问，如：“Customer.bankName”，一般情况下，我们比较推荐用“类名.变量名”的方法访问。而非静态变量是不能用“类名.变量名”的方法访问的。

  • 从底层讲，静态变量在类被载入时创建，只要类存在，静态变量就存在，不管 对象是否实例化。

静态函数

静态变量和静态函数统称为静态成员。静态函数就是在普通函数的定义前加上 static 关键字。

静态函数可以通过“类名.函数名”来访问，当然，也可以通过“对象名. 函数名”来访问。推荐用“类名.函数名”来访问。

静态函数调用时，对象还没有创建，因此，静态函数中不能直接访问类中的非静态成员变量和成员函数。当然，也不能使用 this 关键字。

静态代码块

静态代码块(static block )： 所有对象的共同信息进行初始化，并对所有对象，只执行一次的机制。例如：

java

当类被载入时，静态代码块被执行，且只被执行一次，静态块经常用来进行类属性的初始化。

包

  1. 源代码中，“package chinasei;”表示该源文件中的所有类都位于包 chinasei 中。package 语句必须放在源代码文件的最前面。也可以不指定 package 语句，相当于将类放在缺省包中，不过，指定了包，使用更加方便可靠。
  2. 在 Java 中，我们推荐，包的名字字母一般小写，如“chinasei”、“bank”等， 为了便于阅读，有时候还用“. ”隔开，如：“school.admin”、“school.stu”等。
  3. 将类放入某个包中之后，包将会用专门的文件夹来表示，比如上面的 Customer 类，编译出来的.class 文件路径为：

  1. 如果要用命令行来运行某个包中的类，必须首先到达包目录所在的上一级目录，比如本例中的study目录，然后使用命令： java 包路径.类名

访问包中的类

  1. 在同一个包中，直接用类名来访问，不用指定类所在的包。
  2. 两个类不在同一个包中的情况。

  • 在使用类时，指定类的路径：

    java

  • 用 import 语句导入该类：

    java

  • 如果一个包中类很多，可以用“import 包名.*”来导入该包中的所有类。

  • 有时候，包名中有“.”号，比如：“school.admin”，这并不是说 school 包中，包含了 admin 包。“school.admin”仅仅是一个包名而已。因此，“import school.*;”只 是导入了 school 包中的类，并没有导入 school.admin 包中的类，要导入 school.admin 包中的类，还必须使用“import school.admin.*;”。

访问控制修饰符

  • public 不写的情况下，属于默认访问修饰，此时，该类只能被同一包中的所有类识别。
  • 如果写了 public，该类就是一个公共类，该类可以被包内包外的所有类识别。
  • 如果一个类定义成 public 类，类名和文件名必须相同。因此，一个.java 文件中， 最多只能有一个 public 类。

对于成员来说，访问控制修饰符共有 4 个，分别是private、default、protected、public。default 类型的成员，前面没有任何修饰符。

子类继承父类之后，实例化子类对象的时候，系统会首先实例化父类对象。

  2. 在子类的构造函数中，第一句用 super 来给父类构造函数传参数。

  3. 如果有一些代码是不管异常是否出现都要运行的，用 finally 块将其包起来。

  • Java 规定：一个 try 后面必须至少接一个 catch，可以不接 finally，但是最多只能有一个 finally。

  • 当 try 块中程序出现异常时，try 块中剩余的内容不执行，执行 catch 块，最后执行 finally。

  • 在异常处理机制中，可以加入一个 catch 块来处理其他不可预见的异常。例如:

    java

  • catch(Exception ex)必须写在 catch 块的最后一个，以保证只有前面无法处理的异常，才被这个块处理。

  • 将大量代码放入 try 块，虽然可以保证安全性，但是系统开销较大，程序员务必在系统开销和安全性之间找到一个平衡。

线程(Thread)和进程(Process)的关系很紧密，进程和线程是两个不同的概念，进程的范围大于线程。通俗地说，进程就是一个程序，线程是这个程序能够同时做的各件事情。比如，媒体播放机运行时就是一个进程，而媒体播放机同时做的下载文件和播放歌曲，就是两个线程。因此，可以说进程包含线程。 从另一个角度讲，每个进程都拥有一组完整的属于自己的变量，而线程则共享一个进程内的这些数据。

多线程

第 1 种方法：继承 Thread 类实现多线程：

  1. 编写一个类，继承 java.lang.Thread 类。

  2. 在这个类中，重写 java.lang.Thread 类中的以下函数：

    public void run()

    将线程需要执行的代码放入 run 函数。

  3. 实例化线程对象，调用其 start()函数来启动该线程。

java

  1. 线程的启动，一定要用线程对象的 start()函数，不能用 run()函数。否则就没有多线程的效果。
  2. 主函数的运行也是一个线程，一般称之为主线程。当程序加载到内存时，启动主线程。
  3. 哪个线程先运行完毕，默认情况下由操作系统决定，所以，运行完毕的顺序 不一定是启动的顺序。
  4. 我们可以通过 Thread 的 setPriority(int newPriority)函数来给线程设置优先级，数值越大，优先级越高。

多线程两种方法的区别

继承 Thread 的方法，具有如下特点：

  1. 每一个对象都是一个线程，其对象具有自己的成员变量。
  2. Java 不支持多重继承，继承了 Thread，就不能继承其他类。因此，该类主要完成线程工作，功能比较单一。

实现 Runnable 的方法，具有如下特点：

  1. 每一个对象不是一个线程，必须将其传入 Thread 对象才能运行，各个线程是否共享 Runnable 对象成员，视情况而定。
  2. Java 不支持多重继承，却可以支持实现多个接口，因此，我们有时可以给一些继承了某些父类的类，通过实现 Runnable 的方法增加线程功能

控制线程运行

线程有创建、可运行、运行中、阻塞、死亡五种状态。

  1. 创建状态：使用 new 运算符创建一个线程。
  2. 可运行状态：使用 start()方法启动一个线程后，系统分配了资源。
  3. 运行中状态：执行线程的 run()方法。
  4. 阻塞状态：运行的线程因某种原因停止继续运行。
  5. 死亡状态：线程结束。

线程死锁

产生死锁的四个必要条件是：

  1. 互斥条件：资源每次只能被一个线程使用。如前面的“线程同步代码段”，就是只能被一个线程使用的典型资源。
  2. 请求与保持条件：一个线程请求资源，但因为某种原因，该资源无法分配给它，于是该线程阻塞，此时，它对已获得的资源保持不放。
  3. 不剥夺条件：进程已获得的资源，在未使用完之前，不管其是否阻塞，无法强行剥夺。
  4. 循环等待条件：若干进程之间互相等待，形成一种头尾相接的循环等待资源关系。

可以用如下方法来进行死锁排除：

  1. 可以撤消陷于死锁的全部线程。
  2. 可以逐个撤消陷于死锁的进程，直到死锁不存在。
  3. 从陷于死锁的线程中逐个强迫放弃所占用的资源，直至死锁消失。
