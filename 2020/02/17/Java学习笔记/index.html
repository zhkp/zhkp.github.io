第一章 Java 入门

Java 跨平台的原理

Java 语言编写的源代码，是.java 文件，经过编译后，能够运行的是.class 文件。这些.class 文件不能直接在操作系统中运行，我们需要在不同的操作系统中安装相应的 Java 运行环境(JRE，Java Runtime Environment)，这个 Java 运行环境中包含了 Java 虚拟机(JVM，Java Virtual achine)，.class 文件在不同的 Java 虚拟机上运行即可。

类型转换

  1. 低精度的值可以直接赋值给高精度的变量，直接变成了高精度。

精度：byte

  1. 整型默认为int 即：“byte num = 10;”中10是int，系统隐含做了一个从 int 转向 byte，并不丢失精度的操作，如果不丢失精度，则不报错。上述说明仅仅局限于将常量赋值给变量的情况。当“byte num2 = num + 1;”时就会报错，在变量求值时，byte、short、char 类型的变量值被自动提升为 int 型，最后结果是int，需要强制转换，“byte num2 = (byte) (num + 1);”。

  2. 不同类型变量混合运算之后，得到的结果是精度最高的类型。

比如：

成员函数

Java 中的函数，编写在类中，一般称为成员函数。

如果函数内部的变量和类中的成员变量重名，成员变量被屏蔽。得不到正常效果。此时，可以用“this.”来标志该变量属于类中的成员，而不是局部变量。

函数重载

函数重载，也叫做静态多态。

多态(Polymorphism)，是面向对象编程的特征之一。多态，通俗讲就是一个东西， 在不同情况下呈现不同形态。比如，一个函数名 abs，在不同参数的情况下可以执行不同的代码，而调用者只需要记住一个函数名称。

为什么是静态的呢？那是因为虽然函数名只有一个，但是源代码中，还得根据不同参数编写多个函数。

函数重载必须满足以下条件之一：

  1. 函数参数个数不同。

  2. 函数参数个数相同，类型不同。

  3. 函数参数个数相同，类型相同，但是在参数列表中的出现顺序不同。

构造函数

构造函数也是一种函数，但是定义时必须遵循以下准则：

  1. 函数名称与类相同的名称。

  2. 不含返回类型。

  • 构造函数可以被重载，也就是说，一个类中可以定义多个构造函数。实例化对象时，系统根据参数的不同，调用不同的构造函数。
  • 当一个类的对象创建时，构造函数就会被自动调用，可以在这个函数中加入初始化工作的代码。在对象的生命周期中，构造函数只会调用一次。
  • 在一个类中，如果没有定义构造函数，系统会自动为这个类产生一个默认的构造函数，该函数没有参数，也不做任何事情。因此，我们在没有定义构造函数时，才可以通过“类名 对象名 = new 类名();”来实例化对象。

如果我们自己定义了含有参数的构造函数，系统将不提供默认的构造函数。
