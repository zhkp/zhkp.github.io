8. 字符串转换整数 (atoi) 中等

题目：

  • 请你来实现一个 atoi 函数，使其能将字符串转换成整数。 首先，该函数会根据需要丢弃无用的开头空格字符，直到寻找到第一个非空格的字符为止。
  • 当我们寻找到的第一个非空字符为正或者负号时，则将该符号与之后面尽可能多的连续数字组合起来，作为该整数的正负号；假如第一个非空字符是数字， 则直接将其与之后连续的数字字符组合起来，形成整数。
  • 该字符串除了有效的整数部分之后也可能会存在多余的字符，这些字符可以被忽略，它们对于函数不应该造成影响。
  • 注意：假如该字符串中的第一个非空格字符不是一个有效整数字符、字符串为空或字符串仅包含空白字符时，则你的函数不需要进行转换。 在任何情况下，若函数不能进行有效的转换时，请返回 0。

说明

假设我们的环境只能存储 32 位大小的有符号整数，那么其数值范围为 [−2^31^ , 2^31^ − 1]。如果数值超过这个范围，请返回 INT_MAX (2^31^ − 1) 或 INT_MIN (−2^31^ ) 。

864.一手顺子 中等

题目：

  • 爱丽丝有一手（ hand ）由整数数组给定的牌。
  • 现在她想把牌重新排列成组，使得每个组的大小都是 W ，且由 W 张连续的牌组成。
  • 如果她可以完成分组就返回 true ，否则返回 false 。

提示：

  1. 1 <= hand.length <= 10000
  2. 0 <= hand[i] <= 10^9^
  3. 1 <= W <= hand.length

题解：

  • 这题目直接用数组可解，不过有些麻烦，需要排序并跳过重复，因此我选择使用map进行解答。

  • map key值代表卡牌大小，value值代表这种大小的卡牌的数量。

  • map将key值从小到大排序好，故我们只要每次取map开头元素，然后判断接下来的（W-1）张牌是否连续，否则直接返回Flase；若连续，将这W张牌从map中减去，具体做法为：

    1. 判断卡牌数量是否为1，若是，直接删除该元素；
    2. 若否，则对应元素value值减1；

  • 删去卡牌后，再次取map开头元素，重复第三步，直至map中无元素 。

  • 若前面没有返回Flase，则最后返回True。
