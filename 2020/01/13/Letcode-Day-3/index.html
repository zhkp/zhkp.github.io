835. 图像重叠 中等

题目：

给出两个图像 A 和 B ， A 和 B 为大小相同的二维正方形矩阵。（并且为二进制矩阵，只包含0和1）。

我们转换其中一个图像，向左，右，上，或下滑动任何数量的单位，并把它放在另一个图像的上面。之后，该转换的重叠是指两个图像都具有 1 的位置的数目。

（请注意，转换不包括向任何方向旋转。）

最大可能的重叠是什么？

注意:

  1. ==1 <= A.length = A[0].length = B.length = B[0].length <= 30== ；
  2. ==0 <= A[i] [j], B[i] [j] <= 1==；

题解：

这道题目是我选择动态规划后随机出来的，想了半天也想不到转移方程该怎么写，一看标签，数组，em……这TM，我服了。

明白不是动态规划后，再加上注意里的数据范围，我觉得出题人想要我们用暴力解。

暴力解也有不同的解法，我选择的是移动点，具体算法如下：

记A大小为n*n

  • 初始化结果矩阵result[2 * n] [2 * n] 为 0;
  • 对于A所有的点，每次取一点Aij，先判断该点值是否为一，若否，则跳过再取；
  • 若是，则移动到A中其他位置A(i+k)(j+l)，即判断B(i+k)(j+l)的值，若B(i+k)(j+l)为一，则结果矩阵result[k] [l]++;
  • 每一个值为一的点都要尝试移动所有位置，包括自己原来的位置；
  • 为了方便移动时我首先向右和向下移动，再向左和向上移动；
  • 由于下标无法是负数，故当k或l小于零时，结果保存在对应|k| + n或|l| + n中；
  • 最后再找到结果矩阵中的最大值即为结果；

我还以为暴力破解效率一般，结果前94.41%，是我目前几题里成绩最好的orz

300. 最长上升子序列 中等

题目：

给定一个无序的整数数组，找到其中最长上升子序列的长度。

题解：

终于做到动态规划题了。

这道题算是简单的动态规划了。

假设原数组为num，给定原数组最长上升子序列的长度为N，最长上升子序列末尾元素值为an，往num末尾添加一个元素x，若元素x大于an，则最长上升子序列的长度为N+1；若小于an，则更新上升子序列末尾元素值数组。具体算法如下：

  • 创建vector元素recode，用于记录不同长度末尾元素值。

  • 初始化最长上升子序列的长度result为1，recode[0] = nums[0]；

  • for i from 1 to nums.size - 1

    • if nums[i] > recode[result - 1]

      • recode[result++] = nums[i]

    • else

      • for j from result to 0

        • if recode[j] < nums[i] && recode[j + 1] > nums[i]

          • recode[j + 1] = nums[i]

        • if (recode[0] > nums[i])

          • recode[0] = nums[i];

注意：记得判断空数组。
